## 一、项目概述

### 1.1 项目定位

**Gleamory** 是一个**个人网站首页**，作为一切项目的起点和展示平台。后续所有项目都会以卡片形式挂载在这个首页上。

### 1.2 项目目标

- 提供一个有设计感的项目展示入口
- 集中管理所有项目的展示信息
- 作为个人品牌的统一入口

### 1.3 访问方式

- **当前**: 纯静态页面
- **未来**: 可能扩展后端服务

---

## 二、功能需求

### 2.1 核心功能

#### 功能一：项目卡片展示

**功能描述**: 以卡片形式展示所有项目，点击可跳转到项目详情或在线地址

**卡片信息**:

| 字段 | 必填 | 说明 |
|:---|:---:|:---|
| 项目名称 | ✅ | 项目的显示名称 |
| 项目描述 | ✅ | 一句话描述项目 |
| 项目链接 | ✅ | 点击跳转的 URL |
| 项目状态 | ❌ | 开发中 / 已上线 / 已下线 |
| 项目标签 | ❌ | 多标签，如：在线网站、小游戏、杂项 |
| 项目截图 | ❌ | 封面图，放在 images/ 目录 |
| 项目版本号 | ❌ | 如 v1.0.0 |
| 最近更新时间 | ❌ | 最后一次更新的日期 |

**交互行为**:
- 点击卡片 → 新窗口打开项目链接（不离开首页）
- 没有封面图 → 不显示封面区域

#### 功能二：时间线/动态

**功能描述**: 展示项目的更新动态，让用户了解最新进展

**动态内容**:
- 类型: 项目更新情况（如"xxx项目 v1.0 上线"）
- 格式: 纯文字，无配图
- 展示: 默认最近 10 条，可展开查看全部

### 2.2 项目分类

**标签系统**:
- 支持多标签（每个项目可有多个标签）
- 标签可扩展（后续可添加新类别）
- 暂不做筛选功能（后续可扩展）

**预设标签**:
- 在线网站
- 小游戏
- 杂项

## 三、技术需求

### 3.1 技术栈

| 类别 | 技术选择 |
|:---|:---|
| 框架 | Vue 3 |
| 构建工具 | Vite |
| CSS 框架 | Tailwind CSS |
| 组件 | 自定义组件 |
| 数据存储 | JSON 文件 |

### 3.2 响应式设计

**适配设备**:
- 桌面端（≥1024px）
- 平板端（768px - 1023px）
- 手机端（<768px）

**布局要求**:
- 自动根据屏幕大小调整布局
- 优先保证移动端体验

### 3.3 多语言支持

- **当前**: 仅中文
- **未来**: 可扩展为中英文切换

## 九、设计规范（已确认）

### 9.1 页面标题

- **中文名**: 微光集
- **英文名**: Gleamory
- **展示形式**: `Gleamory 微光集`

### 9.2 配色方案

**主色调**: 粉白配色，梦幻美好的感觉

**参考风格**: 崩坏3角色「人之律者·爱莉希雅」的配色

**配色要素**:
- 主色: 粉色系（柔和的粉、樱花粉、玫瑰粉）
- 辅助色: 白色、米白、浅灰
- 点缀色: 金色、浅紫
- 整体氛围: 梦幻、温暖、优雅

**具体色值（建议）**:
```css
:root {
  --primary-pink: #FFB7C5;      /* 主粉色 */
  --primary-light: #FFF0F5;     /* 浅粉背景 */
  --primary-dark: #E8879E;      /* 深粉色 */
  --accent-gold: #FFD700;       /* 金色点缀 */
  --accent-purple: #E6E6FA;     /* 浅紫点缀 */
  --text-primary: #4A4A4A;      /* 主要文字 */
  --text-secondary: #8A8A8A;    /* 次要文字 */
  --bg-white: #FFFFFF;          /* 白色背景 */
  --bg-cream: #FFFAF0;          /* 米白背景 */
}
```

### 9.3 动画强度

- **当前设置**: 适中
- **后期调整**: 根据实际开发效果微调
- **动画方向**: 柔和、流畅、不突兀
